## STEP WRITING RULES

Write at ERPTM detail level. Each step is one action, one Bizagi interaction, or one verification. Follow these patterns exactly:

Include session codes inline: "Go to Purchase requisition session (tdpur2600m000) and click new."
Include Bizagi form fields: "Open buyer review case. Select RFx required as No, Buyer review completed as Yes. Save and press Next."
Separate verification steps: "Refresh purchase requisition screen. Verify PRQ status is Approved." Do not bury verifications inside a previous step.
Reference PO series type at conversion: 137 for standard/cost/subcontracted, 197 if Return ticked, 397 if CFM Return ticked. Never just "Purchase Order" without a series number.
Reference BPMS case outcomes: "BPMS015 case completed" or "BPMS015 case rejected."

The "data" field is almost always "None". Expected results are short, one statement each.

Step counts: regression tests 10-15 steps, functional rejection tests 5-10 steps.

## TEST TYPE RULES

Functional tests: test the specific change in the ECR. Rejections, validations, edge cases. If a test expects rejection or error, it is ALWAYS Functional.

Regression tests: prove existing happy paths still work. NEVER rejection scenarios. Must go end-to-end: PRQ creation through to PO approved. Always include 2 regression tests: one for the item type being changed, one for a standard unaffected flow.

## KEY PROCESS KNOWLEDGE

BPMS015 = Requisition Approval (Bizagi). Triggers on PRQ submit. Contains buyer review and auto-rejection checks.
BPMS040 = DOA Processing. Triggers when value exceeds buyer's DOA limit. Second user selected, receives email, approves/rejects in Bizagi.
BPMS002.1 = Buyer Work Queue. This is a Bizagi case. Buyer creates it, selects themselves, ticks the PRQ, selects transfer, clicks next. Not just "run BWQ."
BPMS016 = Supplier Response on POs. After PO DOA approval, open PO in Bizagi BPMS016, complete supplier response ACC.

Standard cost item flow:
Create PRQ in tdpur2600m000 > fill header (buyer, business agreement) > create line (item, qty, price, receipt date, BP, pegging info: Project and Activity) > submit > buyer review email with Bizagi case number > open Bizagi, set RFx=No, Buyer review=Yes, Next > DOA if price exceeds limit > PRQ Approved > create BWQ (BPMS002.1) case in Bizagi, select buyer, tick PRQ, transfer > PRQ Converted > find PO via requisition line > References > Linked Requisition Line Data (tdpur2502s000) > open PO in tdpur4100m000 > submit PO > DOA + BPMS016 supplier response ACC > PO Approved

Subcontracted flow (DIFFERENT from standard):
Create production order in tisfc0501m000 > header details (warehouse, routing, dates) > add line with project/activity > Planning Line tab add task (e.g. '341AR') > Actions > Subcontracting > Generate Subcontracting Documents > find PRQ in tdpur2501m000 > maintain PRQ (add Buyer, Purchase office POPS01) > submit > Bizagi buyer review > DOA > PRQ Approved > release production order > BWQ (BPMS002.1) > PRQ Converted > find PO via Linked Requisition Line Data > open PO in tdpur4100m000 > submit > DOA + BPMS016 ACC > PO Approved. Subcontracted PRQs convert to 137 series PO by default.

Item prefixes: ZZZCST = cost items (137 PO). ZZZSCMRO = SCMRO cost items (197 if Return ticked, 397 if CFM Return ticked). ZZZSPL-SUBCON-* = subcontracted items (137 PO).

PO series: 137 = standard/cost/subcontracted (default). 197 = Return Order (Return ticked). 397 = CFM Return Order (CFM Return ticked).

Auto-rejection triggers in BPMS015: missing peg/GI info, blank BP or EMPloyee supplier (rejected after buyer review), SubConDefault items not linked to production orders, multi-line requisitions with subcontracted items.

### Regression: Cost Purchase Requisition - 137 PO Conversion
Summary: "Cost Purchase Requisition - 137 PO Conversion"
Expected result: "Cost purchase requisition approved and converted to 137 series PO successfully"
Steps:
1. Go to Purchase requisition session (tdpur2600m000) and click new. Maintain header with buyer and business agreement -> PRQ header created
2. Create new requisition line. Input cost item (ZZZCST), quantity, price, confirmed receipt date, business partner, pegging information (Project and Activity). Save -> Requisition line created
3. Click on Actions and submit PRQ for approval -> PRQ submitted for approval
4. Check email inbox for buyer review email. Go to Bizagi and input case number from email -> Bizagi case opened
5. Open buyer review case. Select RFx required as No, Buyer review completed as Yes. Save and press Next -> Buyer Review completed. Case sent for DOA approval
6. Complete DOA approval. Refresh Bizagi case and verify case completed with no outstanding actions -> BPMS015 case completed

8. Go to Bizagi and create a BWQ (BPMS002.1) case. Select buyer, tick purchase requisition tick box, click next -> BWQ case created
9. Select the purchase requisition with correct number. Select transfer against that record, click next -> Transfer initiated
10. Refresh the Bizagi case and verify completed with no actions remaining -> BWQ case completed
11. Go back to Infor LN and refresh PRQ. Verify status is Converted. Click requisition line > References > Linked Requisition Line Data (tdpur2502s000) -> PRQ converted. 137 PO number displayed
12. Open purchase order from (tdpur4100m000). Maintain and submit PO for approval -> PO submitted for approval
13. Complete DOA and supplier response. Open PO in Bizagi BPMS016, complete supplier response ACC -> PO is approved
